# PROMPT REPLIT v6 — copy semua di bawah garis ini

---

Rebuild this project **from scratch** as a polished, single-page **interactive presentation website** for **AIGYPT × AINA**, entirely in **Bahasa Indonesia**. It will be shown live on a **giant LED videotron (16:9)** in front of up to **10,000 people** of all backgrounds, most of whom know nothing about AI, as a **30-minute stage talk in 4 chapters ("Bab")**. Afterward it doubles as a normal responsive website.

Two behaviors define this build:
- **ONE press = ONE slide.** The presenter never presses anything to reveal content. Every slide **auto-plays its own content** the moment it appears.
- **Videotron legibility.** Everything must be readable by someone standing 30 meters away from an LED screen. When in doubt: bigger, bolder, fewer words.

## VIDEOTRON RULES (non-negotiable)
1. **Safe area:** all content lives inside a 5% padding on every side — nothing important near the edges (LED rigs often crop edges).
2. **Minimum sizes at 1920×1080:** body text ≥ 26px weight 500; card/label text ≥ 20px weight 600; captions & eyebrows ≥ 16px weight 700; chat bubbles ≥ 20px; team names ≥ 18px weight 700. Headlines: hero `clamp(64px,9vw,150px)`, slide titles `clamp(48px,6.5vw,96px)`, all weight 800.
3. **Contrast:** text is `#FFFFFF` or `#F4F1FA`; secondary text uses `#C9C2E0` (NOT dark gray — dim gray dies on LED). Borders are **2px**, never 1px hairlines. No element below 60% opacity.
4. **Word budget:** max ~40 words visible per slide (chat mockup excluded). Short lines, max ~55 characters per line.
5. Emojis/icons used as visual anchors are large (≥ 34px). Photos are large; grids have generous 20px+ gaps.
6. Squint test: if any element would be unclear from the back row, enlarge or delete it.

## AUTO-CASCADE (replaces click-to-reveal completely)
- When a slide becomes active, its elements animate in **automatically, in a defined order**, staggered **350–450ms apart** (fade + rise 16px, 450ms each). A full cascade finishes within ~3 seconds — except the AINA chat, which types naturally.
- **Keyboard/click does ONE thing only: change slide.** `Space`/`ArrowDown`/`ArrowRight`/click/tap → next slide. `ArrowUp`/`ArrowLeft` → previous. `Home`/`End` → first/last. `F` fullscreen. `T` presenter timer. `R` → replay the current slide's cascade (and chat).
- Re-entering a slide replays its cascade from the start.
- On mobile (<768px): no snap, no cascade delays — content simply appears; site behaves as a normal scrolling page.
- `prefers-reduced-motion`: everything appears instantly.

## Design system
1. **ONE font: "Plus Jakarta Sans"** (Google Fonts; 500, 600, 700, 800). Headlines 800, letter-spacing -0.02em. Eyebrows: 700, uppercase, letter-spacing 0.18em.
2. **Palette:** `--bg:#0B0712` · `--surface:#171126` · `--border:rgba(167,139,250,.28)` · `--accent:#7C3AED` · `--accent-soft:#A78BFA` · `--text:#F4F1FA` · `--muted:#C9C2E0`. Only gradient: `linear-gradient(120deg,#A78BFA,#7C3AED)` — for one key word per headline, primary button, progress bar, and big stat numbers. **No cyan, no magenta, no animated backgrounds.**
3. One decorative element: static radial purple glow behind slide 1 (`radial-gradient(ellipse at 50% 25%, rgba(124,58,237,.25), transparent 60%)`).
4. Max content width 1280px. All animations transform/opacity only.

## Presenter aids
- Fixed chapter label top-center (e.g. `BAB 3 · AINA`) — visible to audience, part of the design, ≥16px weight 700.
- Slide dots right side, grouped per chapter — subtle but visible (10px dots).
- **`T` toggles the presenter HUD:** timer pill `mm:ss / 30:00` (amber at 25:00, red past 30:00) + a small `±N MENIT` label per slide. HUD is **hidden by default** so the videotron stays clean.
- Thin gradient progress bar top.
