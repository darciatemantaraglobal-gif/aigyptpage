# PROMPT UNTUK REPLIT AGENT — copy semua di bawah garis ini

---

Build a single-page **cinematic presentation website** for **AIGYPT × AINA**, entirely in **Bahasa Indonesia**. This site will be projected on a giant LED screen in front of a **10,000-person live audience**, so it must behave like a keyboard-driven stage deck, not a normal scrolling website. It will also be shared as a public URL afterward, so it must still work as a responsive website on mobile.

## Tech stack
- Plain **HTML + CSS + vanilla JS** (single page, no React, no UI libraries, no Tailwind). Speed and zero-dependency reliability matter more than anything: this page must never fail to load on stage.
- Google Fonts only external dependency.
- All images are local files I will upload into an `/images` folder (exact filenames listed below). Do not use stock photos or placeholders anywhere.

## Stage / presentation behavior (most important requirements)
1. The page is divided into **9 full-viewport slides** (each `min-height: 100svh`), using `scroll-snap-type: y mandatory` on desktop.
2. **Keyboard navigation like a slide deck**: `ArrowDown` / `ArrowRight` / `Space` / `PageDown` → next slide; `ArrowUp` / `ArrowLeft` / `PageUp` → previous slide; `Home` / `End` → first / last slide; pressing **F** toggles fullscreen.
3. A thin **gradient progress bar** fixed at the top, plus **slide indicator dots** fixed at the right edge (clickable, with the active dot glowing).
4. **Entry animations replay every time a slide comes into view** (IntersectionObserver): headline rises with a slight blur-to-sharp, supporting elements stagger in 80–120 ms apart. Keep every animation under 700 ms — an arena audience must never wait. Fully respect `prefers-reduced-motion`.
5. **Arena-scale typography**: this is viewed from 100 meters away. Hero headline `clamp(56px, 9vw, 170px)`; slide titles `clamp(40px, 6vw, 100px)`; body text minimum 18px, never more than ~2 short lines per paragraph on desktop. Extreme contrast at all times.
6. On mobile (< 768px): disable scroll-snap, hide the dots, and let it behave as a normal well-designed scrolling landing page.
