Rebuild this project **from scratch** as a clean, single-page **interactive presentation website** for **AIGYPT × AINA**, entirely in **Bahasa Indonesia**. The previous version was cluttered — this version must be minimal, disciplined, and calm. It will be presented live on a giant screen to a **10,000-person audience** as a **30-minute talk**, so it works like a keyboard-driven slide deck with step-by-step reveals, and also works as a normal scrolling website on mobile.

## Non-negotiable design rules (this is what went wrong last time)
1. **ONE font family only: "Plus Jakarta Sans"** (Google Fonts, weights 400, 500, 600, 700, 800). No other typeface anywhere. Headlines = weight 800 with tight letter-spacing (-0.02em). Small labels/eyebrows = weight 600, uppercase, letter-spacing 0.18em, size 12–13px.
2. **Palette: black with a hint of purple. Exactly these tokens, nothing else:**
   - `--bg: #0B0712` (page background)
   - `--surface: #151021` (cards)
   - `--border: rgba(167,139,250,.14)` (all hairlines)
   - `--accent: #7C3AED` and `--accent-soft: #A78BFA`
   - `--text: #F4F1FA` and `--muted: #9D93B8`
   - The ONLY gradient allowed: `linear-gradient(120deg, #A78BFA, #7C3AED)` — used for one highlighted word per headline, the primary button, and the progress bar. No cyan, no magenta, no rainbow, no animated gradient backgrounds.
3. **One decorative element only:** a single large, static radial purple glow behind the hero (`radial-gradient(ellipse at 50% 20%, rgba(124,58,237,.22), transparent 60%)`). Nothing rotating, nothing blurred and animated.
4. Massive whitespace. Max content width 1120px, centered. Consistent vertical rhythm. When in doubt, remove decoration.

## Presentation system (the "interactive" core)
- **10 full-viewport slides** (`min-height: 100svh`), `scroll-snap-type: y mandatory` on desktop, snap disabled below 768px (normal scrolling site on mobile, dots hidden).
- **Keyboard controls:** `Space` / `ArrowDown` / `ArrowRight` / `PageDown` → reveal next fragment, or go to next slide if no fragments remain. `ArrowUp` / `ArrowLeft` / `PageUp` → previous. `Home` / `End` → first/last slide. `F` → fullscreen. `T` → toggle presenter timer. `R` on the AINA slide → replay the chat animation.
- **Fragment system (like PowerPoint builds):** elements marked as fragments start at `opacity:0; transform:translateY(14px)` and reveal one by one on each keypress (transition 400ms). Clicking/tapping a slide also advances fragments. When scrolled past on mobile, all fragments simply show.
- **Presenter timer (T):** a small pill fixed top-left showing elapsed time against a 30:00 target (e.g. `12:41 / 30:00`). Starts on first keypress. Turns amber at 25:00, red past 30:00. Hidden by default so the audience never sees it unless the presenter wants it.
- **Segment label:** tiny muted label fixed bottom-left of each slide, e.g. `SEGMEN 3 · ±5 MENIT` (values given per slide below) — this keeps the 30-minute pacing visible to the presenter at a glance.
- Thin gradient **progress bar** top; clickable **slide dots** fixed right.
- All animations ≤ 500ms, only `transform`/`opacity`. Respect `prefers-reduced-motion` (show everything instantly).
